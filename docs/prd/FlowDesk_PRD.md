## 🎯 产品概述

### 产品背景
用户日常工作流高度依赖 PC（编程、设计、跑 AI 任务、长时间渲染等），但生活节奏碎片化——通勤、外出、临时有事离开工位时，仍需要随时查看 PC 任务进度、临时修改指令、提交下一阶段任务。传统远程桌面软件（ToDesk、向日葵、TeamViewer）虽然功能齐全，但存在以下问题：

1. **注册/登录门槛**：首次使用要注册账号、验证手机、记密码，劝退大量临时用户
2. **连接慢、画质差**：公网中转延迟 100ms+，看不清代码/UI 细节
3. **输入不便**：手机端打字、滑动操作 PC 体验割裂
4. **跨端割裂**：手机复制的代码片段无法快速贴到 PC，反之亦然
5. **隐私顾虑**：账号体系意味着厂商掌握"谁控制谁"的关系图谱

随着 HarmonyOS 6 分布式软总线能力增强，市场上出现了一个**精准的空白**：**鸿蒙手机 → Windows PC 的近场无缝远控，且不需要任何账户**——像蓝牙配对一样简单。

### 产品目标
- **主要目标**：让用户在外 **30 秒内连上自己的 Windows PC**，像坐在电脑前一样完成任务，**全程零注册、零账号**
- **次要目标**：在鸿蒙/Windows 跨生态场景中建立"无账户协同"的心智标杆
- **长期目标**：从远控工具进化为"个人设备协同平台"（消息、文件、任务统一流转）

### 核心价值主张
> **"像配对蓝牙一样，扫码就连上——30 秒连上，3 秒操作，0 账号负担"**
> 鸿蒙手机和 Windows PC 之间的"超近场"远程控制——比 ToDesk 更轻、更快、更私密。

### 产品定位
| 维度 | FlowDesk | ToDesk / 向日葵 | 华为自带"超级终端" |
|---|---|---|---|
| 账户体系 | **❌ 无** | ✅ 强制注册 | ✅ 华为账号 |
| 平台 | 鸿蒙+Windows | 全平台 | 仅华为设备 |
| 局域网延迟 | **<50ms** | 80-150ms | 极低（但功能弱） |
| 公网穿透 | 支持（中转） | 强 | 不支持 |
| 屏幕镜像/控制 | **全功能** | 全功能 | 弱（仅镜像） |
| 文件/剪贴板互通 | **深度集成** | 一般 | 弱 |
| AI 任务场景适配 | **针对 IDE/CLI 优化** | 通用 | 不适用 |
| 隐私 | **服务器零用户数据** | 账号关联 | 账号关联 |

**定位**：聚焦"鸿蒙手机控制 Windows PC"细分场景，做**无账户、轻、快、专**的远控工具。

## 🔧 功能需求

### 核心功能（P0 - MVP 必须实现）

#### 1. 设备身份生成（无账户体系核心）

- **描述**：
  - **Windows 被控端**：首次启动时生成本机唯一 `DeviceID`（格式如 `8K7M-3XQ2-NP5R`，9位分组易记），机器码哈希绑定（重装系统会变，需要重新配对）
  - 同时生成 `AccessKey`（32字节随机数，本地加密存储），用于连接签名
  - **HarmonyOS 主控端**：无需注册，输入对方 `DeviceID` + 完成配对即可
  - 设备身份完全本地生成，**不上传任何 PII（个人身份信息）**
- **优先级**：P0
- **用户价值**：零注册、零手机号、零邮箱
- **验收标准**：
  - 设备ID 全局唯一，碰撞概率 <10⁻¹⁵
  - DeviceID 显示在 PC 端主界面显眼位置，并提供"复制"按钮
  - 重装系统后能识别为"新设备"，旧设备连接会被拒绝

#### 2. 设备发现与配对（替代传统"加好友"）

- **描述**：三种配对方式，按安全级别递减：
  - **A. 扫码配对**（最推荐）：PC 端显示二维码（内含 ECDH 临时公钥 + 设备指纹 + 一次性 Token），手机扫一次完成**永久信任绑定**
  - **B. 动态 PIN 配对**：PC 屏幕显示 6 位数字（每 30 秒刷新，TOTP 风格），手机端输入完成配对
  - **C. 固定密码**（可选）：用户自设 6 位 PIN，作为长期凭证
  - 配对成功后设备互信，无需重复输入
  - 信任关系可手动解除（PC 端"已配对设备列表"中删除）
- **优先级**：P0
- **用户价值**：3 秒完成首次配对，后续直接连
- **验收标准**：
  - 扫码配对成功率 >99%
  - 动态 PIN 错误 5 次后锁定 5 分钟（防爆破）
  - 信任列表上限 16 台（够用）

#### 3. 局域网设备发现

- **描述**：
  - 同一局域网内，手机自动发现 PC（mDNS / 鸿蒙分布式软总线）
  - 设备列表 1 秒内出现，无需输入 IP
  - 跨网段时回退到"输入 DeviceID"模式
- **优先级**：P0
- **用户价值**：同 WiFi 下"开 App 就看到电脑"
- **验收标准**：
  - 同 WiFi 设备发现延迟 <1s
  - 跨网段有清晰提示

#### 4. 屏幕实时镜像

- **描述**：
  - 实时传输 PC 屏幕画面到手机
  - 分辨率自适应（720P/1080P 切换）
  - 帧率自适应（15/30/60fps）
  - 码率自适应（500Kbps ~ 8Mbps）
- **优先级**：P0
- **用户价值**：清晰看到 PC 上 AI 任务输出
- **验收标准**：
  - 局域网延迟 <50ms（1080P@30fps）
  - 公网延迟 <200ms
  - CPU 占用 <15%（PC 端）

#### 5. 触控/键鼠控制

- **描述**：
  - 单指点击 = 鼠标左键
  - 单指拖动 = 鼠标移动 + 按住
  - 双指捏合 = 滚轮
  - 双指点击 = 右键
  - 长按 = 选中
  - 顶部/底部工具栏：键盘、鼠标模式切换、截图、结束
- **优先级**：P0
- **用户价值**：完整操作 PC
- **验收标准**：
  - 操作响应延迟 <80ms
  - 关键 UI 元素（按钮、链接）可精准点击

#### 6. 虚拟键盘

- **描述**：
  - 唤起手机原生输入法
  - 特殊键支持：Ctrl、Alt、Shift、Tab、Esc、方向键、Win 键
  - 代码模式：自动联想常用符号（`{}`、`<>`、`()`）
- **优先级**：P0
- **用户价值**：在 IDE/Trae 中写代码、改 prompt
- **验收标准**：
  - 输入无明显延迟
  - 组合键 100% 准确触发

#### 7. 剪贴板双向同步

- **描述**：
  - 手机复制 → 自动同步到 PC 剪贴板
  - PC 复制 → 通知栏提示"已复制到手机"
  - 支持文本、图片（手机→PC）
- **优先级**：P0
- **用户价值**：搬运代码片段、prompt 模板
- **验收标准**：
  - 同步延迟 <1 秒
  - 中文/特殊字符无乱码

#### 8. 文件快速传输

- **描述**：
  - 手机 → PC：选择文件/图片/视频，一键发送到 PC 指定目录
  - PC → 手机：PC 端可右键文件"发送到手机"（V1.5）
  - 传输走局域网 P2P，不走云端
- **优先级**：P0
- **用户价值**：把手机上拍的照片、文档快速传到 PC
- **验收标准**：
  - 100MB 文件局域网传输 <10 秒
  - 支持断点续传

### 扩展功能（P1 - V1.5）

#### 9. 快捷指令面板
- **描述**：手机端预设常用操作按钮（一键锁屏、关机、打开 Trae/VSCode/浏览器、定时唤醒截图）
- **用户价值**：减少操作路径

#### 10. 后台任务通知
- **描述**：PC 端可订阅特定应用/进程结束事件，完成后推送通知到手机（含屏幕截图）
- **用户价值**：用户在等 Trae AI 跑任务时不用一直盯屏幕
- **注意**：推送走鸿蒙系统 Push 通道，**不需要账号**，使用设备匿名 token

#### 11. 历史会话
- **描述**：保存 7 天内的连接记录，支持一键重连；可回看截图（**仅本地存储，不上传**）
- **用户价值**：方便重复场景

#### 12. 隐私模式
- **描述**：连接时可指定"黑屏"PC 屏幕（防止投屏时被旁人看到）+ 关闭 PC 端扬声器
- **用户价值**：公共场合使用

#### 13. Windows 10 兼容
- 当前 MVP 仅 Win11（DXGI 更好用），V1.5 增加 Win10 支持

### 高级功能（P2 - V2.0+）

#### 14. 公网中转模式
- 自建中转服务器（QUIC 协议）
- **匿名计费**：每台设备首次启动生成匿名 `billing_token`（与 DeviceID 解耦），用于付费
- 订阅制 9.9 元/月
- 服务器**只做流量转发，无法解密内容**

#### 15. 多端互联
- 反向：Windows PC 也能控制鸿蒙手机（屏幕共享）
- 平板作为副屏

#### 16. AI 任务场景深度优化
- 与 Trae、Cursor 等 AI IDE 集成：检测到 AI 输出完成时主动通知
- 支持"任务预设指令"：手机端保存好 prompt 模板，一键发送到 PC 当前输入框

## 👥 用户场景

### 目标用户画像

**主要用户**：
- **画像 A - 鸿蒙手机 + Windows PC 双持用户**（25-40 岁）
  - 拥有华为/鸿蒙手机 + Windows 工作站（**不限于华为电脑**）
  - 日常用 Trae/Cursor/VSCode 写代码，或用 Adobe/Figma 做设计
  - 经常让 AI 跑长任务（代码生成、批量出图）
  - 痛点：跑任务时不敢离开工位，AI 跑完要立刻看结果并追加指令
  - **最在意**：不要注册、不要手机号、要快

- **画像 B - 鸿蒙生态重度用户**（30-50 岁）
  - 华为手机 + 任意品牌 Windows 笔记本（如华硕天选、联想小新等）
  - 工作内容以文档、表格、PPT 为主
  - 痛点：临时外出想拿个文件、改个文档
  - **最在意**：简单、装上就用

**次要用户**：
- 远程办公人群、IT 支持人员、家里帮爸妈调电脑的子女
- **典型场景**：自己临时在父母手机上装一个，输入家里的 DeviceID 即可远程帮父母处理电脑问题

### 典型使用场景

**场景 1：AI 任务进度监控（核心场景）**
- **用户**：鸿蒙开发者，用华硕天选 + 华为 Mate70
- **环境**：下午出门前给 Trae AI 布置了一个长任务（生成一整套组件代码），预计需要 1-2 小时
- **目标**：路上随时查看进度，并在某个节点更新下一步指令
- **流程**：
  1. 出门前：PC 端 FlowDesk 服务已在后台运行（**首次启动时已生成 DeviceID 并显示在桌面小窗**）
  2. 路上：手机下拉通知，看到"AI 任务进行中（已运行 45 分钟）"
  3. 点击通知 → 拉起 FlowDesk → 立即看到 PC 屏幕（**无需登录，直接连接**）
  4. 查看 Trae 输出 → 发现 AI 已完成 70%
  5. 复制手机备忘录里准备好的下一阶段 prompt → 长按粘贴到 Trae 输入框 → 发送
  6. 锁屏退出
- **期望**：整个过程 <2 分钟，**没有任何"请先登录"打断**

**场景 2：临时取文件**
- **用户**：鸿蒙用户
- **环境**：外出开会，老板突然要一个季度报告
- **目标**：从 PC 取文件发到微信
- **流程**：
  1. 手机端 FlowDesk → 看到桌面（**设备列表中点自己的电脑**）
  2. 双击桌面的"2026Q2 报告.docx"打开
  3. 右键 → "发送到手机"
  4. 文件出现在手机相册/下载目录
  5. 微信发送
- **期望**：3 分钟内完成

**场景 3：紧急远程帮父母修电脑**
- **用户**：子女，父母用的是非华为 Windows 笔记本
- **环境**：父母电脑出问题，描述不清楚
- **目标**：远程协助
- **流程**：
  1. 父母在 PC 上打开 FlowDesk，把 DeviceID 念给子女
  2. 子女手机端输入 DeviceID + 父母屏幕上的 6 位动态 PIN
  3. 完成首次配对，后续直接连
  4. 子女远程操作修电脑
- **期望**：5 分钟内建立连接，**不需要父母懂账号/邮箱**

**场景 4：紧急调试**
- **用户**：开发者
- **环境**：服务器出问题，需要立即在 PC 上重启服务
- **目标**：路上用手机完成重启
- **流程**：
  1. 远程连接到 PC
  2. 唤起键盘 → 快捷打开终端
  3. 输入 ssh 命令 + 密码（已保存）
  4. 执行 `systemctl restart nginx`
  5. 查看状态
- **期望**：5 分钟内完成，组合键支持完善

## 🔐 安全模型（无账户体系下的核心）

### 信任建立

```
┌─────────────┐ 扫码/动态PIN  ┌─────────────┐
│  PC 被控端  │◄──────────────►│ 手机主控端  │
│             │  ECDH P-256   │             │
│  生成临时   │  密钥交换       │  验证设备   │
│  公私钥对   │               │  指纹匹配   │
└─────────────┘               └─────────────┘
        │                            │
        └──── 派生共享密钥 ───────────┘
                  ↓
         存储为持久信任凭证
         (加密保存在本地 Keychain)
```

### 加密层级

| 层级 | 算法 | 用途 |
|---|---|---|
| 密钥交换 | **ECDH P-256** | 首次配对，派生共享密钥 |
| 对称加密 | **AES-256-GCM** | 所有业务数据（屏幕流、输入、文件） |
| 完整性 | **HMAC-SHA256** | 防止篡改 |
| 身份认证 | **Ed25519 签名** | 设备 AccessKey 签名连接请求 |
| 密钥派生 | **HKDF-SHA256** | 主密钥 → 媒体密钥 + 控制密钥 + 文件密钥 |
| 防重放 | **时间戳 + Nonce** | 拒绝 60s 外的历史消息 |

### 设备指纹绑定
- PC 端指纹 = SHA256(CPU序列号 + 主板序列号 + 系统盘序列号 + 安装时随机盐)
- 派生密钥时混入设备指纹 → **即使抓包重放，无法在另一台设备上解密**
- 用户可手动重置（"我换电脑了"按钮）

### 威胁模型与防御

| 威胁 | 防御手段 |
|---|---|
| 设备ID 泄露 | 动态 PIN / 固定密码 / 设备指纹绑定 |
| 局域网抓包 | ECDH + AES-256-GCM |
| 中间人攻击 | 首次配对二维码校验指纹 + TLS Pinning |
| 重放攻击 | 时间戳窗口 + Nonce |
| 暴力破解 PIN | 5 次错误锁定 5 分钟 |
| 中继服务器作恶 | 端到端加密，服务器不解密 |
| 物理接触 PC | 信任设备列表可手动撤销 |
| 设备丢失 | 手机端"远程登出"清空本地信任凭证（需重新配对） |

## 📊 成功指标

### 关键指标 (KPI)

**北极星指标**：周活跃连接数（WAU connections）

**用户指标**：
- 7 日留存率 >40%
- 单用户周均连接次数 >5 次
- 单次连接平均时长 >3 分钟
- **首次连接完成率 >85%**（无注册漏斗后应高于账号体系）

**性能指标**：
- 局域网连接建立时间 <5 秒
- 局域网端到端延迟 <50ms（P95）
- 公网端到端延迟 <200ms（P95）
- 月度可用性 >99.5%

**业务指标**（V2.0）：
- 公网功能付费转化率 >5%

### 数据埋点需求
- 连接建立耗时、断开原因
- 配对方式分布（扫码 vs PIN vs 固定密码）
- 屏幕分辨率/帧率分布
- 键盘输入字数、文件传输频次
- **匿名化处理**：所有埋点不带 DeviceID，只带设备型号/系统版本维度

## 🏗️ 技术架构建议

### 客户端架构

**HarmonyOS 6 主控端**：
- ArkTS + ArkUI（声明式 UI）
- Stage 模型
- 分布式软总线 SDK（局域网发现 + P2P 通道）

- .NET 8 + WPF（UI 层）
- DXGI Desktop Duplication API（屏幕采集）
- Media Foundation（硬编 H.264，NVENC/QSV）
- Windows Service 模式（开机自启）
- **DPAPI + TPM 绑定**（本地存储 AccessKey）
- 防火墙自动放行（首次启动引导）

### 协议设计

```
┌─────────────────────────────────────┐
│  应用层：设备管理 / 输入事件 / 文件  │
├─────────────────────────────────────┤
│  安全层：ECDH 握手 + AES-256-GCM    │
├─────────────────────────────────────┤
│  传输层：自定义二进制帧 + Protobuf  │
├─────────────────────────────────────┤
│  媒体层：H.264/H.265 + FEC 纠错     │
├─────────────────────────────────────┤
│  网络层：QUIC（公网）/ UDP（局域网） │
└─────────────────────────────────────┘
```

### 匿名中继服务器（V2.0）

- 设备身份：`DeviceID` + `AccessKey`（与账号完全解耦）
- 流量转发：纯 QUIC relay，**不解密不存储**
- 计费：`billing_token`（首次启动匿名生成，与 DeviceID 无关联）
- 日志：仅保留带宽/连接数等聚合指标，30 天自动清除

### 性能优化
- 屏幕变化区域检测（脏矩形，只传变化部分）
- 自适应码率（ABR，基于网络 RTT）
- 预测渲染（输入预测补偿）
- 硬编硬解优先，软编软解降级

## 📝 备注

### 风险与依赖

1. **Windows 端长期运行稳定性**：需做内存泄漏检测、异常自恢复
2. **无账户带来的滥用风险**：需有连接频率限制、IP 信誉机制（V2.0 中转服务器）
3. **公网中转成本**：V2.0 阶段需要中转服务器，带宽成本是主要支出
4. **DeviceID 遗忘**：用户可能换电脑后忘记原 DeviceID，需提供"我换电脑了"重置流程
5. **隐私合规**：无账户反而更友好，但仍需明确告知用户"正在被远控"和"连接日志本地保存 7 天"

### 后续优化方向
- 与系统级"快速分享"集成（鸿蒙分享菜单直接发送文件到 PC）
- 鸿蒙 PC（如有）原生适配
- AI Agent 自动化场景（手机说一句"把代码提交了"）
- 跨设备光标（手机和 PC 之间拖动文件图标）

### 商业模式建议

无账户方案下，传统订阅/广告转化路径变窄，建议：

- **V1.0**：完全免费（拉新、占心智，建立"无账户远控"品类认知）
- **V1.5**：免费 + 高级功能（多设备管理、历史会话回看等）
- **V2.0**：免费基础 + **公网中转订阅 9.9 元/月**（按设备匿名计费，**不需要账号**）
- **长期**：企业 SaaS（远程运维、技术支持场景，**这是 ToDesk 的核心收入来源**）

**长期商业护城河**：当用户习惯了"无账户、扫码就连"的体验，再也回不去 ToDesk 那种"打开 App 先弹登录"的体验。
